# 公司官网改造设计文档

- 日期：2026-05-13
- 项目路径：`/data/work/web/mycompanysite`
- 设计目标：基于现有 Astro 开源模板，改造成一个中文优先、科技企业风、标准版的公司官方网站，用于品牌展示、综合业务介绍与咨询表单转化。

## 1. 项目背景

当前项目是一个基于 Astro + Tailwind CSS 的开源官网模板，适合用于展示型网站的快速搭建。项目已经具备较完整的页面组织、组件基础、SEO 配置、响应式布局以及部分动效能力，适合作为企业官网改造底座。

用户希望基于该项目快速搭建一个公司的官方网站，而不是完全重做。第一阶段以中文版官网落地为目标，后续再扩展英文版。公司现有内容素材主要来自以下资源：

- 公司介绍资料：`/data/work/web/word/公司手册.docx`
- 公司视频素材：`/data/work/web/video/1bb7b2f447d070e7e3125e77eda3d0d7.mp4`

## 2. 目标与边界

### 2.1 第一阶段目标

第一阶段要交付一个可上线的中文标准版企业官网，满足以下业务目标：

- 展示公司品牌形象
- 展示综合业务能力与服务范围
- 展示解决方案与案例
- 建立可信度
- 引导用户提交咨询表单

### 2.2 非目标

第一阶段不包含以下范围：

- 完整英文版内容落地
- 博客或新闻系统的深度运营化
- 文档中心建设
- 复杂后台管理系统
- CRM 深度集成
- 用户登录系统
- 重型交互应用功能

## 3. 受众与定位

### 3.1 目标受众

本官网需要同时服务多类客户群体：

- 企业管理层与业务负责人
- 技术决策人
- 采购与招投标相关人员
- 政府、国企及大客户单位

### 3.2 网站定位

网站定位为综合型企业官网，既承担品牌展示功能，也承担获客转化与业务说明功能。官网首页需要同时解决以下问题：

- 公司是谁
- 公司做什么
- 为什么值得信任
- 用户下一步该如何联系

### 3.3 视觉与表达定位

整体风格采用科技企业风，要求具备以下特征：

- 深色、现代、专业
- 有一定技术感与未来感
- 信息层次清晰
- 视觉表现克制但不呆板
- 保留并适度强化现有模板中的动效

## 4. 方案选择

本项目采用“在现有开源模板上做内容重构 + 视觉重塑”的方式推进。

### 4.1 采用方案

保留现有项目的技术栈、基础页面结构、组件能力和一部分成熟展示模块，在此基础上完成：

- 企业化内容替换
- 首页重构
- 导航与页脚重构
- 关键内页重构
- 品牌视觉统一
- 表单转化链路建立

### 4.2 不采用的方式

本阶段不做完全重写，也不将当前项目作为纯参考后另起新站。这样可以控制成本、缩短周期，并降低上线风险。

## 5. 页面范围

第一阶段官网包含以下核心页面：

- 首页
- 产品与服务页
- 解决方案页
- 客户案例页
- 关于我们页
- 联系我们页

如果现有模板中存在 blog、docs、products、insights 等结构，则以服务公司官网为准进行复用、改造或删除，不以保留模板原始用途为目标。

## 6. 信息架构设计

### 6.1 顶部导航

一级导航建议如下：

- 首页
- 产品与服务
- 解决方案
- 客户案例
- 关于我们
- 联系我们

其中“联系我们”需要作为主转化入口，在导航、首页 CTA 与页内按钮中重复强化。

### 6.2 首页模块结构

首页建议采用以下顺序：

1. Hero 主视觉区
2. 核心业务能力区
3. 解决方案场景区
4. 公司视频展示区
5. 客户案例或成果摘要区
6. 公司优势区
7. 关于我们摘要区
8. 咨询表单 CTA 区

### 6.3 首页各模块作用

#### Hero 主视觉区

用于在首屏快速传达品牌定位与核心价值，包含：

- 主标题
- 副标题
- 主按钮：立即咨询
- 次按钮：了解解决方案或查看客户案例

首屏必须清楚表达“你们是谁、能做什么、用户下一步该做什么”。

#### 核心业务能力区

以 3 到 6 个卡片说明公司能力矩阵，帮助不同客户快速定位需求入口。内容方向可包含：

- 行业解决方案
- 软件与平台开发
- AI / 智能化能力
- 系统集成与实施
- 运维与技术服务
- 定制化项目交付

#### 解决方案场景区

按行业、场景或痛点组织方案内容，而不是只列服务名称。重点回答“你们如何解决问题”。

#### 公司视频展示区

用于提升真实感、专业感与品牌感知。视频是增强信任的模块，不承担主要信息说明职责。

#### 客户案例区

以 3 到 6 个代表性案例或成果方向建立可信度，重点突出项目类型、交付成果与客户价值。

#### 公司优势区

用简洁条目说明为什么选择该公司，适合强调：

- 综合交付能力
- 快速响应
- 多行业覆盖
- 定制化能力
- 技术与业务协同
- 长期服务能力

#### 关于我们摘要区

从公司手册提炼公司简介与理念，作为关于我们页的入口。

#### 咨询表单 CTA 区

首页底部设置强 CTA 区域，包含标题、说明、表单与补充联系方式，作为首页最终转化出口。

## 7. 内页设计

### 7.1 产品与服务页

按业务板块分组展示能力范围，说明每类服务做什么、适合谁、能带来什么价值。

### 7.2 解决方案页

按行业或场景组织，说明典型痛点、方案内容与交付价值。

### 7.3 客户案例页

展示典型项目或成果方向，重点强调结果与价值，而不是只罗列项目名称。

### 7.4 关于我们页

承载公司介绍、发展理念、核心优势、资质能力与发展愿景。

### 7.5 联系我们页

作为完整转化页，包含：

- 正式咨询表单
- 联系方式
- 地址信息
- 其他可选联系渠道
- 简短沟通说明

## 8. 内容来源与加工策略

### 8.1 内容来源映射

公司手册将作为以下内容的主要来源：

- 公司简介
- 企业定位
- 主营业务
- 解决方案方向
- 典型案例
- 企业优势
- 联系方式
- 愿景、使命、价值观

这些内容将被拆分并映射到首页和各内页，而不是整体照搬原始文档。

### 8.2 视频素材使用方式

公司视频将主要用于：

- 首页品牌展示
- 关于我们页增强展示
- 必要时作为企业展示辅助模块

### 8.3 内容重写原则

所有来自公司手册的内容都需要按官网表达方式进行重写，原则如下：

- 更短
- 更清晰
- 更面向客户
- 更突出价值
- 更适合网页阅读

## 9. 多语言策略

项目最终目标是支持中英双语，但第一阶段只交付完整中文站点。

### 9.1 第一阶段策略

- 中文内容完整落地
- 页面结构避免只适配中文短文案
- 导航、按钮、标题等尽量集中管理，便于未来扩展
- 若现有项目已有合理 i18n 结构，可尽量保留扩展空间

### 9.2 第二阶段策略

后续英文版将基于第一阶段结构补充，不在本阶段实现完整双语内容。

## 10. 表单转化策略

由于官网的首要转化目标是提交咨询表单，因此表单需要形成多点触达。

### 10.1 转化入口布局

- 首页 Hero 主按钮：立即咨询
- 首页中后部 CTA：提交需求或立即咨询
- 联系我们页：正式完整表单
- 解决方案页和案例页可增加辅助 CTA

### 10.2 表单字段建议

第一阶段表单尽量简洁，建议字段如下：

- 姓名
- 公司名称
- 联系方式
- 需求类型
- 需求描述

如需提升线索质量，可视情况增加：

- 行业
- 预算范围
- 项目阶段

但第一版不建议字段过多，以免影响提交率。

### 10.3 表单行为要求

- 提交反馈明确
- 移动端填写顺畅
- 文案强调“我们会尽快联系您”
- 后端实现方式允许先采用轻量方案

## 11. 技术策略

### 11.1 保留技术底座

第一阶段继续使用现有项目技术栈：

- Astro
- Tailwind CSS
- 现有组件机制
- 现有 SEO / sitemap / 构建流程

### 11.2 保留内容

建议保留：

- 基础布局
- 导航与页脚骨架
- 可复用展示组件
- SEO 结构
- 响应式能力
- 一部分动画与视觉过渡
- 合理的多语言扩展结构

### 11.3 优先改造内容

优先改造：

- 首页 Hero 与内容模块
- 导航栏目
- 关于我们内容
- 联系我们与表单
- 页脚信息
- 全站品牌信息
- 视频与静态资源展示

### 11.4 删除或弱化内容

建议删除或弱化：

- 与公司官网无关的 blog 演示内容
- 不必要的 docs 展示内容
- 与业务定位不一致的默认产品页
- 默认品牌文案、图片、社交链接

## 12. 动效策略

用户希望尽量多保留现有模板中的动效，并在关键位置适度加强科技感。

### 12.1 动效保留原则

优先保留或复用以下类型的效果：

- 首屏视觉动效
- 卡片入场动效
- 滚动过渡效果
- 视频展示区过渡
- CTA 区块轻量强调动画

### 12.2 控制原则

动效必须服从以下要求：

- 不影响加载速度
- 不削弱信息可读性
- 不影响移动端体验
- 不干扰表单填写与页面导航
- 不做过度炫技的装饰性动画

## 13. 风险与控制策略

### 13.1 模板感过重

通过重构首页、统一视觉、替换文案和品牌信息解决。

### 13.2 内容结构化不足

通过从公司手册中提炼客户视角文案解决，不直接复制宣传材料。

### 13.3 范围失控

通过只实现标准版官网、中文优先、后置英文版和复杂功能来控制。

## 14. 验收标准

第一阶段完成后，应满足以下标准：

### 14.1 页面可用性

- 首页、产品与服务、解决方案、客户案例、关于我们、联系我们均可访问
- 导航清晰
- 移动端菜单正常使用
- 视频展示正常
- 表单可见、可填写、具备明确提交路径

### 14.2 视觉效果

- 呈现统一科技企业风
- 保留适量动效
- 比原模板更具企业专属性
- 深色、专业、现代、统一

### 14.3 内容质量

- 首页能快速传达品牌、能力与联系路径
- 各内页表达清晰
- 关于我们页能承接公司资料
- 案例页能建立可信度
- 联系我们页能支持转化

### 14.4 工程质量

- 本地开发正常运行
- 构建成功
- 桌面端与移动端浏览正常
- SEO 基础信息已替换
- 无明显死链、占位图、模板残留品牌信息

## 15. 实施顺序建议

1. 梳理现有结构与可复用模块
2. 提取公司手册内容
3. 重构首页
4. 完成核心内页
5. 统一导航、页脚与品牌信息
6. 接入视频与表单
7. 调整动效与视觉细节
8. 完成构建与多端检查

## 16. 结论

本项目第一阶段将基于现有 Astro 开源模板，改造为一个中文优先、科技企业风、标准版的公司官网。网站将围绕品牌展示、综合业务能力说明与咨询表单转化展开，保留现有技术底座与部分成熟动效能力，同时重构首页与核心页面，使其更贴合真实企业官网场景。
