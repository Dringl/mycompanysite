# 公司官网改造设计文档

- 日期：2026-05-13
- 项目路径：`/data/work/web/mycompanysite`
- 设计目标：基于现有 Astro 开源模板，改造成一个中文优先、科技企业风、标准版的公司官方网站，用于品牌展示、综合业务介绍与咨询表单转化。

## 1. 项目背景

当前项目是一个基于 Astro + Tailwind CSS 的开源官网模板，适合用于展示型网站的快速搭建。项目已经具备较完整的页面组织、组件基础、SEO 配置、响应式布局以及部分动效能力，适合作为企业官网改造底座。

用户希望基于该项目快速搭建一个公司的官方网站，而不是完全重做。第一阶段以中文版官网落地为目标，后续再扩展英文版。公司现有内容素材主要来自以下资源：

- 公司介绍资料：`/data/work/web/word/公司手册.docx`
- 公司视频素材：`/data/work/web/video/1bb7b2f447d070e7e3125e77eda3d0d7.mp4`

## 2. 目标与边界

### 2.1 第一阶段目标

第一阶段要交付一个可上线的中文标准版企业官网，满足以下业务目标：

- 展示公司品牌形象
- 展示综合业务能力与服务范围
- 展示解决方案与案例
- 建立可信度
- 引导用户提交咨询表单

### 2.2 非目标

第一阶段不包含以下范围：

- 完整英文版内容落地
- 博客或新闻系统的深度运营化
- 文档中心建设
- 复杂后台管理系统
- CRM 深度集成
- 用户登录系统
- 重型交互应用功能

## 3. 受众与定位

### 3.1 目标受众

本官网需要同时服务多类客户群体：

- 企业管理层与业务负责人
- 技术决策人
- 采购与招投标相关人员
- 政府、国企及大客户单位

### 3.2 网站定位

网站定位为综合型企业官网，既承担品牌展示功能，也承担获客转化与业务说明功能。官网首页需要同时解决以下问题：

- 公司是谁
- 公司做什么
- 为什么值得信任
- 用户下一步该如何联系

### 3.3 视觉与表达定位

整体风格采用科技企业风，要求具备以下特征：

- 深色、现代、专业
- 有一定技术感与未来感
- 信息层次清晰
- 视觉表现克制但不呆板
- 保留并适度强化现有模板中的动效

## 4. 方案选择

本项目采用“在现有开源模板上做内容重构 + 视觉重塑”的方式推进。

### 4.1 采用方案

保留现有项目的技术栈、基础页面结构、组件能力和一部分成熟展示模块，在此基础上完成：

- 企业化内容替换
- 首页重构
- 导航与页脚重构
- 关键内页重构
- 品牌视觉统一
- 表单转化链路建立

### 4.2 不采用的方式

本阶段不做完全重写，也不将当前项目作为纯参考后另起新站。这样可以控制成本、缩短周期，并降低上线风险。

## 5. 页面范围

第一阶段官网包含以下核心页面：

- 首页
- 产品与服务页
- 解决方案页
- 客户案例页
- 关于我们页
- 联系我们页

如果现有模板中存在 blog、docs、products、insights 等结构，则以服务公司官网为准进行复用、改造或删除，不以保留模板原始用途为目标。

## 6. 信息架构设计

### 6.1 顶部导航

一级导航建议如下：

- 首页
- 产品与服务
- 解决方案
- 客户案例
- 关于我们
- 联系我们

其中“联系我们”需要作为主转化入口，在导航、首页 CTA 与页内按钮中重复强化。

### 6.2 首页模块结构

首页建议采用以下顺序：

1. Hero 主视觉区
2. 核心业务能力区
3. 解决方案场景区
4. 公司视频展示区
5. 客户案例或成果摘要区
6. 公司优势区
7. 关于我们摘要区
8. 咨询表单 CTA 区

### 6.3 首页各模块作用

#### Hero 主视觉区

用于在首屏快速传达品牌定位与核心价值，包含：

- 主标题
- 副标题
- 主按钮：立即咨询
- 次按钮：了解解决方案或查看客户案例

首屏必须清楚表达“你们是谁、能做什么、用户下一步该做什么”。

#### 核心业务能力区

以 3 到 6 个卡片说明公司能力矩阵，帮助不同客户快速定位需求入口。内容方向可包含：

- 行业解决方案
- 软件与平台开发
- AI / 智能化能力
- 系统集成与实施
- 运维与技术服务
- 定制化项目交付

#### 解决方案场景区

按行业、场景或痛点组织方案内容，而不是只列服务名称。重点回答“你们如何解决问题”。

#### 公司视频展示区

用于提升真实感、专业感与品牌感知。视频是增强信任的模块，不承担主要信息说明职责。

#### 客户案例区

以 3 到 6 个代表性案例或成果方向建立可信度，重点突出项目类型、交付成果与客户价值。

#### 公司优势区

用简洁条目说明为什么选择该公司，适合强调：

- 综合交付能力
- 快速响应
- 多行业覆盖
- 定制化能力
- 技术与业务协同
- 长期服务能力

#### 关于我们摘要区

从公司手册提炼公司简介与理念，作为关于我们页的入口。

#### 咨询表单 CTA 区

首页底部设置强 CTA 区域，包含标题、说明、表单与补充联系方式，作为首页最终转化出口。

## 7. 内页设计

### 7.1 产品与服务页

按业务板块分组展示能力范围，说明每类服务做什么、适合谁、能带来什么价值。

### 7.2 解决方案页

按行业或场景组织，说明典型痛点、方案内容与交付价值。

### 7.3 客户案例页

展示典型项目或成果方向，重点强调结果与价值，而不是只罗列项目名称。

### 7.4 关于我们页

承载公司介绍、发展理念、核心优势、资质能力与发展愿景。

### 7.5 联系我们页

作为完整转化页，包含：

- 正式咨询表单
- 联系方式
- 地址信息
- 其他可选联系渠道
- 简短沟通说明

## 8. 内容来源与加工策略

### 8.1 内容来源映射

公司手册将作为以下内容的主要来源：

- 公司简介

- 主营业务

- 典型案例
- 企业优势

### 8.2 视频素材使用方式

公司视频将主要用于：

- 首页品牌展示
- 关于我们页增强展示
- 必要时作为企业展示辅助模块

### 8.3 内容重写原则

所有来自公司手册的内容都需要按官网表达方式进行重写，原则如下：

- 更短
- 更清晰
- 更面向客户
- 更突出价值
- 更适合网页阅读

## 9. 多语言策略

项目最终目标是支持中英双语，但第一阶段只交付完整中文站点。

### 9.1 第一阶段策略

- 中文内容完整落地
- 页面结构避免只适配中文短文案
- 导航、按钮、标题等尽量集中管理，便于未来扩展
- 若现有项目已有合理 i18n 结构，可尽量保留扩展空间

### 9.2 第二阶段策略

## 10. 第二阶段：品牌重构与双主题改版

### 10.1 本阶段新增目标

在第一阶段中文官网结构已经落地的基础上，第二阶段聚焦品牌一致性、可读性与内容可信度提升，完成以下目标：

- 修复当前深色主题下多个模块“深底深字”导致的不可读问题
- 建立黑白双主题体系，并保留手动切换能力
- 默认根据用户本地白天 / 夜晚自动匹配主题
- 手动切换后优先使用用户选择，不再被自动规则覆盖
- 用 `/data/work/web/ICON/白色图标.png` 与 `/data/work/web/ICON/黑色图标.png` 替换全站品牌图标
- 用 `/data/work/web/word/公司手册.docx` 与 `/data/work/web/PPT/重庆补全星科技有限公司商业计划书.pptx` 重写官网内容
- 缺失图片时统一替换为更贴近业务的实景行业图
- 在保留现有动效节奏的前提下，弱化模板感并强化企业官网气质

### 10.2 主题系统设计

官网将支持浅色主题与深色主题两套视觉体系。

#### 深色主题

深色主题不再使用接近纯黑的旧配色，而改为更适合企业官网的深石墨灰 / 蓝灰黑体系：

- 页面背景：深石墨灰或蓝灰黑
- 卡片背景：比页面底色略亮一级
- 描边：使用低透明度亮灰或轻微品牌强调色描边
- 标题文字：近白色高对比文本
- 正文文字：中高亮灰色文本
- 次级信息：较弱灰色，但必须保证可读性

#### 浅色主题

浅色主题采用高可读、克制的企业白底体系：

- 页面背景：白色或暖灰白色
- 卡片背景：浅灰或白底卡片
- 标题文字：深灰至接近黑色
- 正文文字：中灰文字
- 按钮与交互强调：使用统一品牌强调色

#### 主题切换逻辑

- 使用导航栏现有的月亮图标位置作为切换入口，不额外增加独立控件
- 第一次访问时，根据用户本地时间自动判断白天 / 夜晚并切换主题
- 白天默认浅色主题，夜晚默认深色主题
- 用户手动切换后，持久化保存主题状态
- 后续访问优先使用手动选择，不再被自动切换覆盖

### 10.3 对比度与可读性修复原则

第二阶段将把“对比度可读性”作为全站级验收要求，而不是只修复单一模块。

重点修复范围包括：

- 首页视频展示区
- 首页业务介绍卡片
- 首页差异化优势区
- 解决方案页卡片与标签区
- 客户案例页封面卡片
- 导航栏与页脚
- FAQ、CTA、联系表单等辅助转化区块

验收标准如下：

- 标题在桌面端与移动端都能一眼辨认
- 正文在深浅主题下都保持稳定可读
- 按钮、输入框、标签与卡片边界不沉入背景
- 切换主题后，信息层级不丢失

### 10.4 图标与品牌素材替换

第二阶段统一替换以下品牌素材：

- 顶部导航 Logo
- 页脚 Logo
- favicon
- apple-touch-icon
- manifest icons
- 社交分享图（OG 图）
- 需要展示品牌识别的局部图标素材

替换策略：

- 深色主题优先使用 `/data/work/web/ICON/白色图标.png`
- 浅色主题优先使用 `/data/work/web/ICON/黑色图标.png`

### 10.5 内容来源策略

第二阶段官网文案改写将采用“双来源整合”方式：

#### 主来源

`/data/work/web/word/公司手册.docx`

用于输出：

- 公司简介
- 三大业务板块
- 技术优势
- 市场优势
- 服务说明
- 业务表达主口径

#### 补充来源

`/data/work/web/PPT/重庆补全星科技有限公司商业计划书.pptx`

用于补充：

- 公司定位
- 品牌理念与使命愿景
- 市场痛点与场景机会

- 重庆 → 西南 → 全国的发展路径
- 丘陵山地市场聚焦逻辑

#### 不直接上官网的内容

以下 PPT 信息不直接进入主官网：

- 融资金额、出让股份、资金用途比例
- 详细财务预测数据
- 强投资人口吻内容

### 10.6 页面级内容重构方向

#### 首页

首页需要重新强调：

- 公司是谁
- 公司在做什么
- 为什么聚焦丘陵山地与智慧农业
- 三大业务板块分别解决什么问题
- 为什么值得信任
- 用户下一步如何发起咨询

首页将重点纳入以下内容：

- 公司定位与一句话价值表达
- 农业社会化服务 / 无人机核心产品 / 无人车平台三大板块入口
- 行业痛点与差异化优势摘要
- 视频展示区高对比重构
- 典型案例或应用场景摘要
- 强 CTA 转化区

#### 产品与服务页

围绕三大业务板块重写，突出：

- 做什么
- 服务谁
- 带来什么价值
- 与传统方式相比的优势

#### 解决方案页

按场景组织内容，而不是按抽象能力组织。重点覆盖：

- 丘陵山地农业场景
- 规模化农业场景
- 公共服务场景（巡检、消杀、园区等）
- 从需求调研到方案落地的实施流程

#### 客户案例页

如果暂时没有适合公开披露的实名客户案例，则采用“典型场景案例”写法，强调：

- 场景
- 问题
- 方案
- 成效

避免伪造具体客户名称或不具备公开依据的数据。

#### 关于我们页

重点引入以下内容：

- 公司简介
- 品牌使命与愿景
- 聚焦丘陵山地的原因
- 三位一体业务生态
- 重庆起步、西南拓展、全国布局的发展路径

#### 联系我们页

强化企业咨询转化表达，支持：

- 产品采购咨询
- 场景合作咨询
- 渠道合作咨询
- 商务合作与项目咨询

### 10.7 图片替换策略

图片替换遵循以下优先级：

1. 若 Word 或 PPT 中存在可直接使用的真实公司图片，优先使用真实素材
2. 若无真实素材，则统一使用“实景行业图”替代模板图
3. 仅在必要时保留少量抽象科技感图片作为气氛辅助，不得喧宾夺主

缺失图片时优先补充以下类型：

- 农业机械化与农田实景作业图
- 无人机植保与巡检作业图
- 无人车、巡检、园区、公共服务场景图
- 贴近重庆丘陵山地农业环境的场景图

## 10. 表单转化策略

由于官网的首要转化目标是提交咨询表单，因此表单需要形成多点触达。

### 10.1 转化入口布局

- 首页 Hero 主按钮：立即咨询
- 首页中后部 CTA：提交需求或立即咨询
- 联系我们页：正式完整表单
- 解决方案页和案例页可增加辅助 CTA

### 10.2 表单字段建议

第一阶段表单尽量简洁，建议字段如下：

- 姓名
- 公司名称
- 联系方式
- 需求类型
- 需求描述

如需提升线索质量，可视情况增加：

- 行业
- 预算范围
- 项目阶段

但第一版不建议字段过多，以免影响提交率。

### 10.3 表单行为要求

- 提交反馈明确
- 移动端填写顺畅
- 文案强调“我们会尽快联系您”
- 后端实现方式允许先采用轻量方案

### 11.4 删除或弱化内容

建议删除或弱化：

- 与公司官网无关的 blog 演示内容
- 不必要的 docs 展示内容
- 与业务定位不一致的默认产品页
- 默认品牌文案、图片、社交链接

## 12. 动效策略

用户希望尽量多保留现有模板中的动效，并在关键位置适度加强科技感。

### 12.1 动效保留原则

优先保留或复用以下类型的效果：

- 首屏视觉动效
- 卡片入场动效
- 滚动过渡效果
- 视频展示区过渡
- CTA 区块轻量强调动画

### 12.2 控制原则

动效必须服从以下要求：

- 不影响加载速度
- 不削弱信息可读性
- 不影响移动端体验
- 不干扰表单填写与页面导航
- 不做过度炫技的装饰性动画

## 13. 风险与控制策略

### 13.1 模板感过重

通过重构首页、统一视觉、替换文案和品牌信息解决。

### 13.2 内容结构化不足

通过从公司手册中提炼客户视角文案解决，不直接复制宣传材料。

### 13.3 范围失控

通过只实现标准版官网、中文优先、后置英文版和复杂功能来控制。

## 14. 验收标准

第一阶段完成后，应满足以下标准：

### 14.1 页面可用性

- 首页、产品与服务、解决方案、客户案例、关于我们、联系我们均可访问
- 导航清晰
- 移动端菜单正常使用
- 视频展示正常
- 表单可见、可填写、具备明确提交路径

### 14.2 视觉效果

- 呈现统一科技企业风
- 保留适量动效
- 比原模板更具企业专属性
- 深色、专业、现代、统一

### 14.3 内容质量

- 首页能快速传达品牌、能力与联系路径
- 各内页表达清晰
- 关于我们页能承接公司资料
- 案例页能建立可信度
- 联系我们页能支持转化

### 14.4 工程质量

- 本地开发正常运行
- 构建成功
- 桌面端与移动端浏览正常
- SEO 基础信息已替换
- 无明显死链、占位图、模板残留品牌信息

## 15. 实施顺序建议

1. 梳理现有结构与可复用模块
2. 提取公司手册内容
3. 重构首页
4. 完成核心内页
5. 统一导航、页脚与品牌信息
6. 接入视频与表单
7. 调整动效与视觉细节
8. 完成构建与多端检查

## 16. 结论

本项目第一阶段将基于现有 Astro 开源模板，改造为一个中文优先、科技企业风、标准版的公司官网。网站将围绕品牌展示、综合业务能力说明与咨询表单转化展开，保留现有技术底座与部分成熟动效能力，同时重构首页与核心页面，使其更贴合真实企业官网场景。
